Showcase journey

Money flow

1

Top up

Create a PromptPay request. Xendit's payment webhook credits spending exactly once.

2

Fund work

Creating a job moves employer spending into held-for-jobs atomically.

3

Earn

A normal synthetic worker applies, submits, and receives earnings after approval.

4

Payout

Earnings are reserved, sent through Payouts V2, then finalized or released by webhook.

Product boundary: there is no generic user-to-user transfer. Value reaches a worker only after an employer approves funded work. Disputes are deferred but their schema was retained.

Code organization

Main modules

Frontend

web/src/App.vue is the five-step journey. api.ts handles canonical envelopes and redacted logging. job-flow.ts normalizes selected-worker identity.

Auth and actors

src/modules/auth/ owns Google/Better Auth and KU email policy. src/modules/dev-test/ owns synthetic users and opaque actor sessions.

Money and payments

src/modules/money/ exposes wallet/activity and accepts webhooks. src/modules/payments/ creates Xendit top-ups and payouts.

Funded jobs

src/modules/jobs/ implements fund, apply, select, submit, approve, and atomic settlement.

Correctness

Money invariants

  • Whole-baht API balances; provider fee/tax snapshots may use satang.
  • Four user compartments: spending, earnings, held-for-jobs, reserved-for-payouts.
  • Every movement is a balanced, sealed ledger transaction.
  • Policy and fee snapshots preserve historical meaning.
  • Mutations use idempotency keys and stable request hashes.
  • Provider webhooks—not create responses—are authoritative for final value.
  • Webhook intake authenticates, persists, deduplicates, acknowledges, then drains with retry.

docker compose down -v
docker compose up -d postgres

Do not regress

Security handoff

  • Secrets and callback tokens are deliberately absent from this page.
  • Rotate any secret copied into chat, tickets, or screenshots.
  • Keep constant-time webhook authentication, CSRF origin checks, idempotency, and database ledger constraints.
  • Do not expose full bank accounts, QR strings, raw callback payloads, cookies, or tokens in browser diagnostics.
  • The synthetic actor feature must remain development-only.
  • Replace Quick Tunnels with a stable HTTPS domain and managed secrets before production.
